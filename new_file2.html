Webpack资源模块化管理和打包工具


一、建立空的文件夹，在文件夹内打开终端，输入npm init 即可


相关名词的解释:

(1)package name 项目名称，不分大小写

(2)version 默认为1.0.0

(3)description 项目描述

(4)entry point 项目入口文件，目前这阶段不需要设置

(5)test command 测试命令，目前不需要设置

(6)git repository 按照真实情况填写，目前不填写

(7)keywords 关键字

(8)author 作者


项目建立完成之后，会在项目文件夹内生成package.json文件，这是我们项目的配置文件


二、在你的项目文件夹内手动建立一个webpack.config.js的文件，作为webpack插件的配置文件



三、安装 webpack 和webpack-cli


(1)两个都安装完成后在 终端输入 webpack -v 看下是否安装成功，有版本号出现说明安装成功。


(2)安装插件
通过使用 npm install --save-dev 插件名 安装， 这个指令安装可以直接添加到写入package.json文件。


四、 配置webpack的入口和出口


(1)进入webpack.config.js的文件里配置


//第一步，引入系统路径 const path=require('path');


//第二步，


module.exports={

	//在这里写入口，出口的代码。
}
完整的配置代码如下: 
 module.exports={

	//在这里写入口，出口的代码。

	// 入口：可以自己规定从哪里进

	entry:  '你选择的入口文件路径' ,   //   ，命名为entry.js

              //其中出口分为两个部分:
    
	output:{

   	      path:path.resolve(__dirname,'dist') ,  //这部分是固定的
    
    	      filename:'文件名字需要带后缀',      //一般设置为bundle.js
	}
	//还有解析不可识别的文件后缀
	module:{
	         rules:[{
	                test:/.css$/,
		//解析css需要什么loader
		use:['style-loader','css-loader']
		},

		{
		test:/.less$/,
		use:['style-loader','css-loader','less-loader']
		}
	        ]
		
	    }
       }

五、设置入口文件（entry.js）的内容


//引入需要打包的文件


require(‘文件路径’)

//到这里以上都配置好 最后在终端执行webpack，即可生成出口文件。
